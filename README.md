GhostStep 协议与 GhostKey Mini 概念构想

**【成本核算（BOM 估算）】**
主控：STM32F0 系列 MCU —— 约 ¥6
存储：EEPROM (AT24Cxx) —— 约 ¥2
交互：物理拨动开关 + Type-C 接口 —— 约 ¥3
安全：硬件 RTC 时钟芯片 —— 约 ¥4
PCB 及阻容件：SMT 贴片与小批量打板 —— 约 ¥10~15

总计：约 ¥25~¥30（量产可压至 ¥20 以内）

**一句话介绍：**
一种低成本（30 元级）、物理开关否决、离线根种子派生的物联网控制体系。同时可作为硬件级网络信任锚，为网站访问提供抗抓包、防设备失窃的安全凭证。

核心外设为 GhostKey Mini：带 EEPROM 与物理拨动开关的 USB-C 硬件密钥，
本身内嵌中继器逻辑，插手机后拨开关生成单次凭据，再发往执行器（如 LED）。

**核心设计点（本人提出）：**
1. 信任根离线化：根种子仅存于 GhostKey 内部 EEPROM，永不导出到手机或网络。
2. 物理否决权：所有控制必须经过 GhostKey 上物理拨动开关闭合，开关断则全链失效。
3. 确定性派生：每次凭据 = HMAC-SHA256(根种子, 计数器)，用完计数器加一，一次一密。
4. 候选池混淆：每次发送 N 个候选凭据（1 真 N-1 假），接收端恒定时间验证，抗抓包分析。
5. 中继功能内嵌：GhostKey 本身承担第一跳信任锚与生成器，无需部署多个分布式中继器。
6. 急停降级：日常不用物理开关，感知异常时拨开关断电断网；维修时整体下电。
7. 成本极致：STM32F0 + EEPROM + 拨动开关 + Type-C，BOM 估算 30 元级。
8. 抗全频带阻尼降级：无线被压时退化为硬连线或红外链路，无线模块可物理断电。

**安全机制补充说明（防时序攻击与容错策略）**

1. 恒定时间验证（防时序攻击）接收端对本次收到的 N 个候选凭据（1 真 N-1 假）逐一执行：
- 计算期望凭据：Expect = HMAC(Seed, Counter)
- 恒定时间比较：
  diff = 0
  for k in 0..31: 
      diff |= Expect[k] ^ Cand_i[k]
- 不提前退出，循环固定 32 字节。仅当 diff==0 且位置匹配则认证通过。
- 目的：攻击者无法通过响应耗时微差推断真密码字节，杜绝 Timing Side-Channel（时序侧信道）。



2. 计数器同步策略（容错窗口）
- **默认模式（高安全）**：严格一次一密（W=0），错一个号即拒绝，需物理复位。
- **可用模式（高便利）**：允许验证窗口 `Counter ∈ [C, C+W]`，建议 W=2。
  - 验证成功后，将本地计数器置为成功值 +1。
  - **严禁**验证 `Counter < C`（防止重放攻击）。
- **注意**：W 是可用性参数，非密码学强度变量。本构想默认 W=0。



**威胁模型与对应：**
- 手机被恶意 APP 控制：截获本次动态密码，但无根种子、无物理开关，下次凭据失效。
- 抓包：只看到候选池噪声，恒定时间无侧信道，无法逆推根种子。
- 硬件被拆：EEPROM 读取需物理拆解，根种子高熵，暴力破解成本远超目标价值。
- 维修态：断网断电，RAM 清空，无动态攻击面。

**与现有协议区别：**
不取代 OAuth/HTTPS，定位为高安全场景的补充控制协议；
以“人参与物理动作 + 离线派生”换取便利性，不以无感为设计目标。

**未来用途（含机械方向）：**
数控机床权限启动、实验设备授权、智能家居高安全控制、机电一体化安全子模块。

**补充：网络安全与防攻击说明**

此前介绍多以控制 LED 等物理外设为例，容易让人误解其能力仅限于本地控制。实际上，GhostKey Mini 的架构从设计之初便充分考虑了网络环境，其防御逻辑更为严苛。

对抗设备失窃

即便手机或电脑被盗，GhostKey Mini 内部的根种子依然无法被导出。攻击者面对的是一个无法读取核心密钥的硬件黑盒。由于凭据通过单向函数生成，即使拿到设备，也无法窃取你的网络身份。

对抗流量分析（防抓包）

设备生成的网络凭据包含一个混淆候选池，且服务器采用恒定时间算法进行验证。这意味着，黑客即便抓包成功，也无法通过响应速度的差异分辨出真正的密码，更无法通过单向函数反推源头密钥。

简而言之，它能控制一盏灯，更能守住网络的大门。物理安全与网络安全，本就同源
